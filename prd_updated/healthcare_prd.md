## 2. 프로젝트 배경

일반 사용자는 건강검진 결과를 가지고 있어도 BMI, 혈압, 혈당, 콜레스테롤 수치가 어떤 의미인지 해석하기 어렵다. 또한 건강수치 확인 이후 어떤 생활관리 행동으로 이어져야 하는지 알기 어렵다.

특히 젊은 사용자는 혈압·혈당·콜레스테롤 등 일부 건강정보를 모르는 경우가 많다. 따라서 모든 건강정보를 필수로 요구하는 서비스보다는, 사용자가 알고 있는 최소 건강정보만으로도 기본 결과를 제공하고 추가 입력값에 따라 결과를 고도화하는 서비스 구조가 필요하다.

본 프로젝트는 공공 건강검진 데이터를 활용해 사용자의 입력값을 비교군과 비교하고, 건강관리 유형을 분류하며, 생활관리 방향과 지역사회 건강관리 자원을 안내하는 자가점검형 헬스케어 웹앱을 구현한다.

## 3. 문제 정의

| 문제 | 설명 |
| --- | --- |
| 건강수치 해석 어려움 | 사용자가 BMI, 혈압, 혈당, 콜레스테롤 수치의 의미를 이해하기 어려움 |
| 입력 장벽 | 혈압·혈당·콜레스테롤을 모르는 사용자가 많음 |
| 행동 연결 부족 | 수치 확인 이후 어떤 행동을 해야 할지 불명확함 |
| 정보 분산 | 보건소·건강생활지원센터 등 지역자원 정보 접근이 어려움 |
| 과잉진단 리스크 | 건강관리 안내가 질병 진단처럼 보일 위험이 있음 |

## 5. 타깃 사용자

| 사용자군 | 주요 니즈 | 서비스 가치 |
| --- | --- | --- |
| 20~30대 일반 사용자 | 건강검진 결과가 없거나 건강위험 인식이 낮음 | BMI·생활습관 중심 기본 자가점검 |
| 30~40대 직장인 | 건강검진 결과는 있으나 수치 해석이 어려움 | 혈압·혈당·콜레스테롤 기반 관리 방향 제시 |
| 50대 이상 사용자 | 혈압·혈당·체중 관련 관리 방향 확인 | 복합관리형 분류와 관리 포인트 제공 |
| 건강검진 결과 보유자 | 본인 수치를 입력해 비교군 기반 결과 확인 | 입력값 기반 비교군 비교 |

## 6. 핵심 사용자 시나리오

| 단계 | 사용자 행동 | 시스템 반응 | 비고 |
| --- | --- | --- | --- |
| 1 | 서비스에 접속한다 | 서비스 목적, 데이터 출처, 주의문구를 표시한다 | |
| 2 | 성별, 연령대, 키, 몸무게를 입력한다 | BMI를 자동 계산하고 기본 건강지표를 준비한다 | 필수 입력 |
| 3 | 혈압, 공복혈당, 총콜레스테롤, 허리둘레를 선택 입력한다 | 입력 완성도 산출 및 룰 기반 해석 준비 | 선택 입력 |
| 4 | 흡연, 음주, 운동 빈도·강도·시간을 입력한다 | 생활관리 추천 문구를 보완한다 | 권장 입력 |
| 5 | 거주지역을 선택한다 | 해당 지역의 보건기관 및 건강생활지원센터 정보를 검색한다 | 선택 입력 |
| 6 | 결과를 확인한다 | 건강관리 유형, 비교군 비교, 주요 관리 포인트, 생활관리 추천을 출력한다 | |
| 7 | 지역사회 자원을 확인한다 | 보건소·보건지소·건강생활지원센터 정보를 카드 형태로 표시한다 | |

## 7. 서비스 컨셉

### 7.1 핵심 컨셉

사용자가 알고 있는 건강정보만 입력해도 건강관리 방향을 알려주는 자가점검 서비스

### 7.2 서비스 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| 입력 | 사용자가 성별, 연령대, 신체정보, 검진수치, 생활습관, 지역을 입력 |
| 룰 기반 해석 | BMI, 혈압, 혈당, 콜레스테롤 기준에 따라 참고 문구 출력 |
| 데이터 기반 분석 | 국민건강보험공단 건강검진정보를 활용해 건강관리 유형 도출 |
| 비교군 비교 | 동일 성별·연령대 검진자 그룹과 주요 지표 비교 |
| 추천 | 건강관리 유형과 입력값에 따라 생활관리 방향 제안 |
| 지역자원 안내 | 지역보건의료기관과 건강생활지원센터 정보 제공 |

### 7.3 서비스가 하는 것과 하지 않는 것

| 하지 않는 것 | 하는 것 |
| --- | --- |
| 질병 진단 | 건강관리 참고용 자가점검 |
| 의료 처방 | 생활관리 방향 제안 |
| 건강보험공단 개인자료 자동 연동 | 사용자가 직접 입력 |
| 식단·영양제 추천 | 만성질환 예방관리 중심 추천 |
| 혈액검사 필수 입력 | 입력 가능한 정보 수준별 결과 제공 |
| 입력값 저장 | 세션 기반 일회성 분석 |
| 운동 정보를 모델 학습에 사용 | 운동 입력값은 생활관리 추천 보완에만 사용 |

## 8. 기능 요구사항

### 8.1 입력값 설계

핵심은 사용자가 모르는 값을 입력하지 않아도 서비스가 작동하게 만드는 것이다.

| 입력 단계 | 항목 | 필수 여부 | 결과 수준 |
| --- | --- | --- | --- |
| 기본 입력 | 성별, 연령대, 키, 몸무게 | **필수** | BMI 기반 기본 건강관리 유형 |
| 생활습관 입력 | 흡연, 음주, 운동 관련 항목 | **권장** | 생활관리 추천 보완 |
| 혈압 입력 | 수축기혈압, 이완기혈압 | **선택** | 혈압관리형 판단 |
| 검진결과 입력 | 공복혈당, 총콜레스테롤, 허리둘레 | **선택** | 대사복합관리형 판단 |
| 지역 입력 | 시도 또는 시군구 | **선택** | 건강생활지원센터·보건소 안내 |

### 8.2 기능 요구사항 목록

| 기능 ID | 기능명 | 설명 | 우선순위 |
| --- | --- | --- | --- |
| F-01 | 기본정보 입력 | 성별, 연령대, 거주지역을 입력받는다 | Must |
| F-02 | 신체정보 입력 | 키, 몸무게를 입력받고 BMI를 계산한다 | Must |
| F-03 | 건강검진 수치 입력 | 혈압, 공복혈당, 총콜레스테롤, 허리둘레를 선택 입력받는다 | Must |
| F-04 | 모름 처리 | 미입력 항목이 있어도 결과를 제공한다 | Must |
| F-05 | 생활습관 입력 | 흡연, 음주, 운동 빈도·강도·시간을 입력받는다 | Should |
| F-06 | 룰 기반 건강지표 해석 | BMI·혈압·혈당·콜레스테롤 기준에 따라 참고 문구를 출력한다 | Must |
| F-07 | 건강관리 유형 분류 | 모델 기반으로 사용자의 건강관리 유형을 분류한다 | Must |
| F-08 | 비교군 비교 | 동일 성별·연령대 검진자 그룹과 주요 지표를 비교한다 | Should |
| F-09 | 생활관리 추천 | 건강관리 유형별 생활관리 문구를 출력한다 | Must |
| F-10 | 운동 추천 보완 | 운동 빈도·강도·시간에 따라 신체활동 관련 추천을 보완한다 | Should |
| F-11 | 지역사회 자원 안내 | 보건소·보건지소·건강생활지원센터 정보를 표시한다 | Should |
| F-12 | 주의문구 표시 | 진단·치료·처방이 아님을 명시한다 | Must |

| 2 | 변수 선택 | 성별, 연령대, 시도, 키, 몸무게, 허리둘레, 혈압, 혈당, 콜레스테롤, 흡연, 음주 |
| 3 | 전처리 | 결측치 처리, 이상치 제거, 범주형 인코딩 |
| 4 | 파생변수 생성 | BMI, 혈압관리 지표, 혈당관리 지표, 체중관리 지표, 대사지표 개수 |
| 5 | 룰 기반 기준 설계 | BMI·혈압·혈당·콜레스테롤 기준별 안내 문구 |
| 6 | 군집화 | K-Means 또는 Gaussian Mixture |
| 7 | 군집 해석 | 군집별 평균 BMI, 혈압, 혈당, 흡연율 등 비교 |
| 8 | 분류모델 학습 | 사용자 입력값 → 건강관리 유형 예측 |
| 9 | 추천 로직 설계 | 군집별 생활관리 문구 매핑 |
| 10 | 운동 추천 로직 설계 | 사용자 운동 빈도·강도·시간 기반 문구 매핑 |
| 11 | Streamlit 구현 | 입력 페이지, 결과 페이지, 지역자원 페이지 |

## 11. 건강관리 유형 설계

건강관리 유형은 분석자가 완전히 임의로 정하는 것이 아니다. 공공 건강검진 데이터로 군집화한 뒤, 각 군집의 평균값과 위험요인 분포를 해석해 사람이 이름을 붙이는 방식으로 설정한다.

| 건강관리 유형 | 주요 특성 | 추천 방향 |
| --- | --- | --- |
| 기본관리형 | BMI·혈압·혈당이 상대적으로 정상 범위 | 정기검진 유지, 건강생활 실천 |
| 체중관리형 | BMI 또는 허리둘레 관리 필요 지표가 두드러짐 | 걷기, 식사량 점검, 체중 모니터링 |
| 혈압관리형 | 수축기·이완기혈압 관리 필요 지표가 두드러짐 | 혈압 재측정, 저염식, 운동, 의료기관 상담 권고 |
| 혈당관리형 | 공복혈당 관리 필요 지표가 두드러짐 | 식습관 점검, 건강검진 재확인, 상담 권장 |
| 대사복합관리형 | 체중·혈압·혈당·콜레스테롤 중 복수 지표 관리 필요 | 보건소 상담, 의료기관 상담, 지속관리 권고 |
| 생활습관관리형 | 흡연·음주 관련 요인이 두드러짐 | 금연·절주·신체활동 프로그램 안내 |

### 11.1 유형명 설계 원칙

| 비추천 표현 | 추천 표현 |
| --- | --- |
| 대사복합위험형 | 대사복합관리형 |
| 혈압위험형 | 혈압관리형 |
| 혈당위험형 | 혈당관리형 |
| 비만위험형 | 체중관리형 |
| 흡연음주위험형 | 생활습관관리형 |

## 12. 결과 출력 정책

### 12.1 출력 영역

| 출력 영역 | 내용 |
| --- | --- |
| 입력 완성도 | 입력한 항목 기준 35%, 60%, 85% 등 표시 |
| 기본 건강지표 | BMI, 체중 상태 |
| 룰 기반 1차 해석 | BMI, 혈압, 혈당 등 기준별 참고 안내 |
| 건강관리 유형 | 체중관리형, 혈압관리형, 혈당관리형, 대사복합관리형 등 |
| 주요 관리 포인트 | 체중, 혈압, 혈당, 흡연, 음주, 신체활동 점검 등 |
| 비교군 비교 | 공공 건강검진 표본 내 동일 성별·연령대 비교군의 평균 특성 |
| 추천 행동 | 혈압 재측정, 걷기, 금연, 절주, 건강검진 확인 |
| 지역사회 자원 | 건강생활지원센터, 보건소, 보건지소 등 |
| 주의문구 | 의학적 진단이 아닌 참고용 안내 |

### 12.2 결과 표현 원칙

| 기존 표현 (사용 금지) | 수정 권장 표현 |
| --- | --- |
| 당신의 유형은 "대사복합위험형"입니다. | 입력값 기준, 당신은 "대사복합관리형"에 가까운 것으로 나타났습니다. |
| 혈압관리 필요 가능성 | 혈압 재측정 및 상담 권장 |
| 혈당관리 필요 가능성 | 공복혈당 재확인 및 상담 권장 |
| 생활습관 조정으로 개선 가능성이 있습니다. | 입력된 수치상 생활습관 관리 방향을 함께 점검해볼 필요가 있습니다. |

### 12.3 룰 기반 출력 기준

| 항목 | 기준 예시 | 출력 문구 예시 |
| --- | --- | --- |
| BMI | 25 이상 | 체중관리가 도움이 될 수 있습니다. |
| 허리둘레 | 남성 90cm 이상, 여성 85cm 이상 | 복부비만 관련 관리가 필요할 수 있습니다. |
| 혈압 | 수축기 140 이상 또는 이완기 90 이상 | 혈압 재측정 및 상담을 권장합니다. |
| 공복혈당 | 100~125 | 공복혈당 관리가 필요할 수 있습니다. |
| 공복혈당 | 126 이상 | 가까운 의료기관에서 확인검사를 권장합니다. |
| 총콜레스테롤 | 200~239 | 지질관리 참고가 필요할 수 있습니다. |
| 총콜레스테롤 | 240 이상 | 지질검사 결과 확인 및 상담을 권장합니다. |

## 19. 이론적 배경: 건강신념모형(Health Belief Model)

본 프로젝트는 건강신념모형(Health Belief Model)을 이론적 기반으로 적용한다. 건강신념모형은 개인이 건강행동을 실천하는 과정에서 위험 인식, 심각성 인식, 행동의 이익, 행동의 장애, 행동계기, 자기효능감이 중요하다고 보는 모형이다.

| 건강신념모형 요소 | 프로젝트 적용 |
| --- | --- |
| 지각된 민감성 | 내가 어떤 건강관리 유형에 가까운가? |
| 지각된 심각성 | 이 위험요인이 왜 관리가 필요한가? |
| 지각된 이익 | 혈압 측정, 걷기, 금연, 검진이 왜 도움이 되는가? |
| 지각된 장애 | 혈압·혈당을 몰라도 기본 결과를 제공 |
| 행동계기 | 보건소·건강생활지원센터·검진 확인으로 연결 |
| 자기효능감 | 이번 주 실천 가능한 작은 행동 제안 |
