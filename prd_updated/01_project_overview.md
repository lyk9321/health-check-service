## 1.2 한 줄 설명

사용자가 알고 있는 건강정보를 입력하면, 국민건강보험공단 건강검진 공공데이터에서 도출한 건강관리 유형과 비교해 **맞춤형 생활관리 방향**을 추천하는 자가점검형 헬스케어 웹앱.

---

## 1.3 배경 및 필요성

### 1.3.1 사회적 맥락
- 국내 만성질환(고혈압·당뇨·이상지질혈증) 유병률은 30대 이상 성인의 30% 이상에 달하나, 건강검진 결과를 제대로 해석하는 사람은 소수
- 건강검진 수검률은 지속 상승하지만 결과 활용률은 낮음 → 수치를 보고도 "어떻게 하면 되는지" 모르는 사람이 다수
- 특히 20~30대는 혈압·혈당 등 기본 수치 자체를 모르는 경우가 많아, 기존 건강관리 서비스의 접근성이 낮음

### 1.3.2 기술적 기회
- 국민건강보험공단 건강검진정보 2024년 100만 명 표본이 공공데이터로 공개 → 추가 비용 없이 대규모 실데이터 활용 가능
- K-Means 군집화 + Random Forest 분류모델로 사용자 건강관리 유형을 실데이터 기반으로 분류 가능
- Streamlit을 활용하면 Python 분석 코드를 빠르게 웹앱으로 서비스화 가능

### 1.3.3 포트폴리오 포지셔닝

| 구분 | 내용 |
|------|------|
| 서비스 형태 | 사용자가 직접 건강정보를 입력하면 결과가 나오는 Streamlit 웹앱 |
| 핵심 데이터 | 국민건강보험공단 건강검진정보 2024년 100만 명 표본 |
| 핵심 분석 | BMI, 혈압, 혈당, 콜레스테롤, 흡연·음주 등 기반 건강관리 유형 분류 |
| 핵심 모델 | 군집화 (K-Means) + 분류모델 (Random Forest) + 룰 기반 추천 |
| 포트폴리오 포지션 | 간호 지식을 가진 데이터 분석가가 만든 개인 맞춤 건강관리 MVP |

## 1.4 프로젝트 범위 (Scope)

### 1.4.1 In Scope

| 항목 | 설명 |
|------|------|
| 데이터 전처리 파이프라인 | 건강검진 데이터 결측치 처리, 파생변수 생성, 표준화 |
| 군집화 분석 | K-Means로 건강관리 유형 도출, 군집 해석 및 유형명 부여 |
| 분류 모델 학습 | Random Forest로 사용자 입력값 → 건강관리 유형 분류 |
| 룰 기반 건강지표 해석 | BMI·혈압·혈당·콜레스테롤 기준에 따른 참고 문구 출력 |
| 비교군 비교 | 동일 성별·연령대 검진자 그룹 대비 주요 지표 비교 |
| 생활관리 추천 | 건강관리 유형별 생활관리 문구 및 운동 추천 |
| Streamlit 웹앱 구현 | 모바일 우선 1열 카드형 UI |
| 지역사회 자원 안내 | 보건소·보건지소·건강생활지원센터 연동 |
| 모델 평가 | Accuracy, F1-Score, Confusion Matrix, Silhouette Score |
| 산출물 | Jupyter Notebook 4종 + Python 모듈 5종 + app.py + README + 발표자료 |

### 1.4.2 Out of Scope

| 항목 | 이유 |
|------|------|
| 질병 진단 및 의료 처방 | 의료법·윤리적 리스크 |
| 건강보험공단 개인자료 자동 연동 | 보안·인증·API 승인 문제 |
| 로그인·회원가입·사용자 데이터 저장 | 개인정보 처리 부담 |

## 1.5 기술 접근법 요약

### 1.5.1 두 가지 판단 레이어

```
Layer 1 (룰 기반 · 즉시 해석)
──────────────────────────────────────
사용자 입력값 → BMI 계산
→ BMI / 혈압 / 혈당 / 콜레스테롤 기준표 비교
→ 항목별 참고 문구 출력

Layer 2 (데이터 기반 · 유형 분류)
──────────────────────────────────────
사용자 입력값 → src/predict.py
→ Random Forest 분류모델 (model/classifier_model.pkl)
→ 건강관리 유형 예측
→ src/recommend.py → 유형별 생활관리 추천 문구
→ data/processed/comparison_group_summary.csv → 비교군 비교
```

### 1.5.2 기술 선택 근거

| 기술 | 대안 | 선택 이유 |
|------|------|-----------|
| Streamlit | Flask, FastAPI, Dash | Python 분석 코드와 직결, 개발 속도, 무료 배포 |
| K-Means | DBSCAN, GMM | 해석 가능성, 군집 수 제어 용이 |
| Random Forest | XGBoost, LogReg | 해석 가능성(변수 중요도), 적은 데이터에도 안정적 |
| pandas + scikit-learn | PySpark, PyTorch | 100만 건 수준은 pandas로 충분, 생태계 표준 |

## 1.7 전제 조건 (Assumptions)

1. 국민건강보험공단 건강검진정보 2024년 공공데이터는 프로젝트 시작 전 다운로드 완료
2. Python 3.10 이상, 일반 노트북(CPU) 환경에서 동작 (GPU 불필요)
3. Streamlit Community Cloud 무료 플랜으로 배포 (512MB RAM 제한 고려)
4. 운동 정보는 건강검진 데이터에 없으므로 모델 학습에는 사용하지 않음
5. 서비스는 교육용·참고용이며 의료 진단을 대체하지 않음
