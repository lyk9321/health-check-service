## 2.1 핵심 문제 정의

### 2.1.1 현재 상황 (As-Is)

```
건강검진 결과지 수령
      │
      ▼
[사용자 혼자 해석]  ← "BMI 25.3이면 뭐가 문제야?"
      │
      ▼
  의미 모름  ← 수치의 기준, 비교 대상, 행동 방향 불명확
      │
      ▼
  결과지 방치  ← 연간 수검자의 절반 이상이 결과 미활용
```

**문제점**:
1. **비해석성**: BMI·혈압·혈당 수치를 봐도 "정상 범위인지" "어느 정도 심각한지" 판단 불가
2. **입력 장벽**: 혈압·혈당·콜레스테롤을 모르는 20~30대는 기존 서비스 자체를 이용 불가
3. **행동 연결 부족**: 수치 확인 후 "그래서 뭘 해야 하지?"에 대한 안내가 없음
4. **비교 기준 부재**: "나와 비슷한 사람들은 어떤지" 맥락 정보 없이 숫자만 나열
5. **지역자원 정보 분산**: 보건소·건강생활지원센터 정보가 어디 있는지 알기 어려움

### 2.1.2 목표 상황 (To-Be)

```
사용자 건강정보 입력 (아는 것만)
      │
      ▼
[룰 기반 1차 해석]  ← BMI/혈압/혈당 기준 즉시 안내
      │
      ▼
[데이터 기반 건강관리 유형 분류]  ← 100만 명 실데이터 기반
      │
      ▼
[비교군 비교]  ← "동일 성별·연령대 검진자 그룹 대비 OO 수준"
      │
      ▼
[맞춤 생활관리 추천 + 지역자원 안내]  ← 구체적 다음 행동 제시
```

## 2.2 건강관리 유형 수학적 정의

### 2.2.1 기본 표기법

| 기호 | 정의 |
|------|------|
| $X \in \mathbb{R}^{n \times p}$ | 건강검진 데이터 행렬 (n: 표본 수, p: 변수 수) |
| $x_u \in \mathbb{R}^p$ | 사용자 입력 건강정보 벡터 |
| $\tilde{x}_u$ | 표준화된 사용자 입력 벡터 |
| $\hat{y}_u \in \{\text{basic, weight, blood\_pressure, blood\_sugar, metabolic, lifestyle}\}$ | 사용자 건강관리 유형 예측값 |

### 2.2.2 BMI 계산

$$\text{BMI} = \frac{\text{체중 (kg)}}{\text{키 (m)}^2}$$

**BMI 판정 기준** (대한비만학회 기준):

| 범위 | 분류 | 출력 문구 |
|------|------|----------|
| < 18.5 | 저체중 | 체중 증가 관리가 도움이 될 수 있습니다. |
| 18.5 ~ 22.9 | 정상 | 현재 체중 범위는 적정 수준입니다. |
| 23.0 ~ 24.9 | 과체중 | 체중 관리를 점검해볼 필요가 있습니다. |
| ≥ 25.0 | 비만 | 체중관리가 도움이 될 수 있습니다. |

### 2.2.3 비교군 정의

동일 성별($g$) × 연령대($a$)에 해당하는 건강검진 표본 집합:

$$\mathcal{G}_{g,a} = \{x_i \in X \mid \text{gender}_i = g,\ \text{age\_group}_i = a\}$$

**비교군 표시 기준**:

| 조건 | 표시 방식 |
|------|----------|
| $|\mathcal{G}_{g,a}| \geq 1{,}000$ | 정상 비교군 비교 제공 |
| $500 \leq |\mathcal{G}_{g,a}| < 1{,}000$ | 참고용으로만 표시 |
| $|\mathcal{G}_{g,a}| < 100$ | 비교군 비교 미제공 |

### 2.2.4 건강관리 유형 라벨 부여 (룰 기반)

임상 기준에 따라 우선순위 규칙을 적용하여 각 검진자에게 건강관리 유형 라벨을 부여합니다.

$$
\hat{y}_i = \text{Rule}(x_i) =
\begin{cases}
\text{metabolic}       & \text{if } \text{risk\_count} \geq 3 \\
\text{blood\_pressure} & \text{else if } \text{sbp} \geq 130 \text{ or } \text{dbp} \geq 80 \\
\text{blood\_sugar}    & \text{else if } \text{fbg} \geq 100 \\
\text{weight}          & \text{else if } \text{bmi} \geq 25 \text{ or abdominal obesity} \\
\text{lifestyle}       & \text{else if } \text{smoking} = \text{현재흡연} \text{ or drinking} = 1 \\
\text{basic}           & \text{otherwise}
\end{cases}
$$

- 우선순위 순서: 복합위험 → 혈압 → 혈당 → 체중 → 생활습관 → 기본
- 하나의 검진자는 반드시 하나의 유형에만 속함
- 이 라벨을 정답(y)으로 삼아 Random Forest 분류 모델을 학습함

### 2.2.5 사용자 건강관리 유형 분류

학습된 Random Forest 분류기 $f$:

$$\hat{y}_u = f(\tilde{x}_u)$$

- $\tilde{x}_u$: 사용자 입력값을 학습 데이터 scaler로 표준화한 벡터
- 결측 입력 변수는 해당 성별·연령대 평균값으로 대체 후 별도 안내 문구 표시
- 분류 기준은 2.2.4의 룰 기반 라벨 정의와 일치함 (학습 데이터 라벨 = 임상 기준)

## 2.3 건강관리 유형 6단계 상세 정의

| 유형 | 라벨 | 주요 특성 | 추천 방향 | 우선 추천 자원 |
|------|------|---------|----------|--------------|
| 기본관리형 | `basic` | BMI·혈압·혈당 정상 범위 | 정기검진 유지, 건강생활 실천 | 건강생활지원센터 |
| 체중관리형 | `weight` | BMI 또는 허리둘레 관리 필요 | 걷기, 식사량 점검, 체중 모니터링 | 건강생활지원센터·보건소 |
| 혈압관리형 | `blood_pressure` | 수축기·이완기혈압 관리 필요 | 혈압 재측정, 저염식, 의료기관 상담 | 보건소·의료기관 |
| 혈당관리형 | `blood_sugar` | 공복혈당 관리 필요 | 식습관 점검, 건강검진 재확인 | 보건소·의료기관 |
| 대사복합관리형 | `metabolic` | 체중·혈압·혈당·콜레스테롤 복수 관리 필요 | 보건소 상담, 지속관리 권고 | 보건소·의료기관 |
| 생활습관관리형 | `lifestyle` | 흡연·음주 관련 요인 두드러짐 | 금연·절주·신체활동 프로그램 | 금연클리닉·건강생활지원센터 |

> **유형명 원칙**: "위험형"이 아닌 **"관리형"** 표현을 우선 사용하여 과잉 진단 오해 방지

## 2.4 문제의 어려움 (Technical Challenges)

### Challenge 1: 클래스 불균형

```
룰 기반 라벨링 후 실제 데이터 분포 (추정):
기본관리형    : ████████████████████ 40%
체중관리형    : █████████████        30%
혈압관리형    : ██████               15%
혈당관리형    : ████                 10%
대사복합관리형 : ██                   4%
생활습관관리형 : █                    1%

→ 해결: 분류 시 class_weight='balanced' 적용
        Macro F1-Score를 주 평가 지표로 사용
```

### Challenge 2: 결측 입력값 처리

```
문제: 혈압·혈당·콜레스테롤을 모르는 사용자
→ 해결: 해당 변수는 성별·연령대 평균으로 대체
        단, 출력 시 "OO 정보 미입력으로 해당 항목 평가 제한" 문구 표시
        입력 완성도 % 지표로 사용자 피드백 제공
```

### Challenge 3: 클래스 라벨 불균형

```
문제: 룰 기반 라벨링 시 기본관리형·체중관리형에 데이터가 집중되고
      혈당관리형·대사복합관리형·생활습관관리형은 소수 클래스가 됨
→ 해결: Random Forest의 class_weight='balanced' 옵션으로 자동 보정
        소수 클래스에 더 높은 가중치를 부여하여 학습 불균형 완화
        평가 시 Macro F1-Score를 주지표로 사용하여 소수 클래스도 동등하게 반영
```

### Challenge 4: 표현 리스크 관리

```
문제: 건강관리 안내가 "진단"처럼 보일 위험
→ 해결: "OO관리형에 가깝습니다" 표현 사용
        모든 출력 하단에 의료적 고지 문구 필수 포함
        "고혈압입니다" 등 단정적 표현 금지
```

### Challenge 5: 운동 데이터 부재

```
문제: 건강검진 데이터에 운동 변수 없음
→ 해결: 운동 관련 입력은 별도 룰 기반 모듈로 처리
        주간 신체활동량 계산 → 권장 수준과 비교 문구 출력
        모델 학습 변수에는 절대 포함하지 않음
```

## 2.5 가정 및 제약 사항

### 2.5.1 유효한 입력 조건
- 성별: 남성 / 여성 (이진 선택)
- 연령대: 10대~80대+ (10세 단위)
- 키: 100cm 이상 250cm 이하
- 몸무게: 20kg 이상 300kg 이하
- 혈압: 수축기 60~250mmHg, 이완기 40~150mmHg
- 공복혈당: 50~500mg/dL
- 지질 수치 (4종 모두 선택 입력, 아는 항목만 입력 가능)
  - LDL 콜레스테롤: 10~300mg/dL
  - HDL 콜레스테롤: 10~150mg/dL
  - 트리글리세라이드(중성지방): 10~2,000mg/dL
  - 총콜레스테롤: 50~500mg/dL (LDL 미입력 시 보조 참고 지표로 사용)

### 2.5.2 현재 버전 미지원 케이스
- 임산부 (별도 기준 필요)
- 18세 미만 (소아·청소년 기준 상이)
- 건강검진을 한 번도 받지 않은 사용자 (기본 BMI 기준으로만 제공)

### 2.5.3 모델 출력의 한계
- 건강관리 유형은 통계적 유사군 분류이며 의료 진단이 아님
- 인접 유형(예: 체중관리형 ↔ 대사복합관리형) 오분류 가능성 있음
- 입력값이 적을수록(완성도 낮을수록) 분류 신뢰도 저하 가능
